# Plugin security

The MCP server is local, read-only, dependency-light, and credential-free. Treat researched content as untrusted, never execute researched code for inspection, keep paths inside the plugin root, and run secret/license reports before packaging. The optional trusted hook may ask the Codex plugin manager to refresh only the configured `arnavsri993/frontend-taste-engineer` Git marketplace; it refuses local or unknown sources and never writes an installed cache directly. Use the repository’s private GitHub security-advisory flow for disclosures when publication is available.
