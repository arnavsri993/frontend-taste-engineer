# Source license gates

This is an engineering provenance policy, not legal advice. Evaluate the exact item being used; a site, repository, package, documentation page, example, font, icon, screenshot, and brand asset can have different terms.

## Intended-use matrix

| Evidence state | Code copy | Adapted implementation | Inspiration/reference |
|---|---|---|---|
| Clear permissive license and obligations recorded | Allowed after notices, dependency, security, accessibility, and project-fit review | Allowed with applicable notice/attribution and verification | Allowed with attribution when required |
| Paid/proprietary with verified entitlement and public terms | Only within the entitlement and redistribution limits | Only within the entitlement and derivative-work limits | Link/reference within terms; do not redistribute catalog content |
| Website visible but license unstated | Blocked | Blocked | Generalized observations only; no screenshots/assets/text redistribution |
| Root license but embedded asset/brand scope unclear | Code may be allowed; unclear items blocked | Code may be adapted; unclear items blocked | Link and create original replacements |
| Inaccessible, unresolved, or injection-flagged | Blocked | Blocked | Review record only; do not rely on substantive claims |
| Inspiration-only gallery | Blocked | Blocked | Generalized multi-source patterns only |

## Required record

Store the owner, canonical and supplied URLs, exact public surfaces consulted, revision/date, license text or URL, intended use, attribution/notice duties, paid/account boundaries, third-party asset exceptions, dependency license implications, classification, restrictions, reliability, and recheck condition.

## Premium and pro sources

A subscription, account, purchase, client-side HTML, copy button, or “free” label does not imply redistribution rights. Confirm the user/project entitlement and exact terms before use. Never place paid/proprietary source material in plugin knowledge, eval fixtures, screenshots, packages, or public artifacts.

## Brands and case studies

Do not copy logos, trademarks, screenshots, testimonials, names, marketing copy, product data, or distinctive page expression merely because a source is public or open source. OpenAI Build Week and other corporate/product/event marketing pages are never pullable catalogs; they can be user-supplied case-study evidence only.

## Promotion

An unclear license always remains `unresolved`, `inaccessible`, `inspiration-only`, or `rejected`; scoring cannot override the gate. Promotion must update `research/license-review.md` and retain source-specific restrictions in the registry and any derived guidance.
