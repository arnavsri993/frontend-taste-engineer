# Monthly source discovery

1. Start from explicit coverage gaps and unstable framework/browser areas.
2. Search primary standards, vendor/framework documentation, original case studies, and maintained design systems.
3. Record candidates with authority, originality, specificity, actionability, evidence, maintenance, production relevance, accessibility/performance implications, license, generalizability, duplication, and trend/aesthetic risk.
4. Classify as core, specialized, experimental, inspiration-only, rejected, inaccessible, or deprecated.
5. Write a candidate report and registry proposal. Do not modify stable knowledge.
6. Require human review, license review, and evaluation before promotion.
