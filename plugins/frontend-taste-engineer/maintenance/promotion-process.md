# Candidate promotion

1. Create a focused branch from `candidate`.
2. Add registry/provenance changes and candidate records with conflicts and restrictions.
3. Run schema, provenance, duplicate, contradiction, coverage, retrieval, and frontend evaluations.
4. Compare against the previous stable plugin across representative cases, not one showcase.
5. Require mandatory-rule recall and provenance correctness to avoid regression.
6. Open a pull request into `candidate`; resolve review and evaluation gaps.
7. Promote from `candidate` to `main` only through a second reviewed pull request.
8. Update version, changelog, packages, and release report. Never auto-merge.
